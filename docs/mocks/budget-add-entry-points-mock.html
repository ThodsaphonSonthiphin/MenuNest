① ปุ่ม + Cat (สีชมพู) ที่ group header แตะแล้วเปิด AddCategoryDialog เดิม — pre-fill group ของแถบที่กด ผู้ใช้แค่กรอกชื่อ + emoji + target (ถ้าอยาก) → กด Create. sortOrder ไม่ต้องกรอก (auto จาก backend T3)
② ปุ่ม ＋ Add Group (สีเขียว) ท้าย list เปิด AddGroupDialog ใหม่ที่ต้องสร้าง — มีแค่ field "Name" อย่างเดียว. หลังสร้าง group ใหม่จะมาอยู่ต่อท้าย (sortOrder auto จาก backend T1)
③ แตะที่ RTA hero (สีเหลือง) เปิด SetIncomeDialog ใหม่ที่ต้องสร้าง — field "Monthly income for May 2026" + ค่าปัจจุบัน. กด Save → ใช้ useSetMonthlyIncomeMutation ที่มีอยู่แล้ว → RTA จะอัปเดตทันที. มี icon ✎ มุมขวาบนของ hero ให้เห็นว่ากดได้
④ เมนู ⋯ บน /budget/accounts/:id → "Reconcile balance" (สีฟ้า) เปิด ReconcileBalanceDialog ใหม่ — ให้ user กรอก "ยอดจริงจากธนาคาร" (real bank balance). ระบบคำนวณส่วนต่างจากยอดที่ track ไว้แล้วสร้าง transaction "Reconcile adjustment" (categoryId = null, amount = diff, notes = "Manual balance fix"). Backend ใช้ useCreateBudgetTransactionMutation ตัวเดิม — ไม่ต้องเพิ่ม backend ใหม่. transaction adjustment จะเห็นในรายการพร้อม icon ⚖ ให้รู้ที่มา. เมนูเดียวกันยังมี "Edit account" + "Close account" (สำหรับ UpdateAccount / Close ที่มีอยู่แล้วใน backend)
