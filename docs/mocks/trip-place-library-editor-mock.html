Feature — คลังสถานที่ + Place profile (master data ข้ามทริป)
คลิกการ์ดในแท็บ คลังสถานที่ → เปิดหน้าต่างนี้ แก้ 3 ฟิลด์ที่เป็นของสถานที่ (ช่วงเวลาที่ดีที่สุด · ลิงก์รีวิว · สิ่งที่ต้องเตรียม) — ตัด dwell/การเดินทาง/เวลาถึง-ออก ที่เป็นของ "จุดแวะ". ข้อมูลไหลผ่าน Place profile ระดับผู้ใช้ (master, key = user + Google place_id): capture → seed จาก master, แก้ในทริป = override เฉพาะทริป, enrich ครั้งแรก → สร้าง master อัตโนมัติ, ครั้งถัดไปกด "ดันขึ้น master" เอง.

1 · แท็บคลังสถานที่ — คลิกการ์ดเพื่อเปิด

2 · หน้าต่างแก้ไขสถานที่ (เปิดจากคลัง) — seed มาจาก master

โมเดลข้อมูล (master + per-trip override):
  • Place profile (master) — ระดับผู้ใช้ key (UserId, GooglePlaceId) เก็บ best-time + review links + ชุดรายการเช็คลิสต์ (ไม่เก็บสถานะ "เตรียมแล้ว")
  • capture ทุกทริป → seed ค่าจาก master ลง TripPlace (ถ้ามี master)
  • บันทึก = เขียนแค่ TripPlace (override เฉพาะทริป); enrich ครั้งแรกที่ยังไม่มี master → สร้าง master อัตโนมัติ
  • ดันขึ้น master = opt-in เอาค่าปัจจุบันของทริปนี้ไปทับ master → capture ครั้งหน้าได้ค่านี้
  • เอาออกจากทริปนี้ = ลบ TripPlace (เช็คลิสต์ entry cascade) — master ไม่หาย = "soft delete" ที่ต้องการ
  • ไม่มี place_id → ไม่มี master (ทำงานเหมือนเดิม) · MCP: capture ก็ seed จาก master, ปุ่มดันขึ้น master = Phase 2 · หน้าจอจัดการ master ทั้งคลัง = Phase 2
